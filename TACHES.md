## Transition manette Xbox → manette NATHAN

C'est un jalon majeur du projet. Grâce aux interfaces, le basculement est un **swap d'implémentation** :

```
AVANT (S6 → S8 sem 5)                    APRÈS (S8 sem 5+)
┌─────────────────┐                       ┌─────────────────┐
│   Game Engine   │                       │   Game Engine    │
│  (inchangé)     │                       │  (IDENTIQUE)     │
│                 │                       │                  │
│  InputMapper ───┤→ GamepadMapper        │  InputMapper ────┤→ NathanMapper
│                 │  (manette PS4/Xbox)   │                  │  (joysticks ADC + GPIO)
│                 │                       │                  │
│  HapticEngine ──┤→ GamepadHaptic        │  HapticEngine ───┤→ NathanHaptic
│                 │  (1 moteur manette)   │                  │  (5 moteurs PWM)
│                 │                       │                  │
│  AudioEngine ───┤→ PCAudio              │  AudioEngine ────┤→ EmbeddedAudio
│                 │  (miniaudio sur PC)   │                  │  (I2S + HRTF sur MCU)
│                 │                       │                  │
│  Storage ───────┤→ PCStorage            │  Storage ────────┤→ SDCardStorage
│                 │  (fichiers locaux)    │                  │  (FAT32 sur micro SD)
└─────────────────┘                       └─────────────────┘
```

**Conditions pour effectuer la transition :**
- [ ] PCB soudé et testé électriquement (S8-T01)
- [ ] NathanMapper implémenté et testé sur breadboard (S8-T04)
- [ ] NathanHaptic implémenté et testé (S8-T04)
- [ ] Audio embarqué fonctionnel sur le MCU (S8-T02, S8-T05)
- [ ] Boîtier imprimé et assemblé (S8-T03)

**La transition ne se fait pas d'un coup** — elle peut être incrémentale :
1. D'abord swapper Input seul (NathanMapper) → jouer au jeu PC avec la vraie manette
2. Puis swapper Haptic (NathanHaptic) → sentir les 5 moteurs
3. Puis swapper Audio (EmbeddedAudio) → tout tourne sur le MCU
4. Enfin swapper Storage (SDCardStorage) → console autonome

### Goulots d'étranglement

| Goulot | Pourquoi c'est critique | Impact si retard |
|--------|------------------------|------------------|
| **Benchmark PC** (S7 sem 6-8) | Bloque le choix MCU → bloque TOUT le hardware | Retarde le PCB, la manette, le portage, les tests |
| **Choix du MCU** (S7 sem 8-9) | Point de convergence des flux A, B et C | Aucun travail matériel ne peut avancer sans cette décision |
| **Fabrication PCB** (2-3 sem incompressibles) | Délai externe non contrôlable | Si le PCB arrive en retard, la transition manette est repoussée |
| **Portage audio embarqué** (S8 sem 1-6) | Le plus complexe techniquement — HRTF en temps réel sur MCU | Si trop lent : fallback stéréo simple ou DSP externe |
| **Approbation CER** (>= 4 sem avant test) | Délai administratif non contrôlable | Soumettre dès S7 pour avoir la marge |
